1. 네이밍 규칙

1. 기본 규칙

1. 일반 규칙 : 이름은 영문 소문자, 숫자, 언더스코어(_)로 작성한다.

2. 시작 이름 : 소문자로 시작한다.

3. 영문 소문자, 숫자, 언더스코어 조합 : pageWrap, ban_t, page1 등

2. id 및 class 네이밍 규칙

1. id, class - id는 문서 전체의 고유 식별자로 한 문서에서 동일한 id를 여러 번 사용하지 않는다. - class는 문서에서 여러 번 사용할 수 있다.

2. 레이아웃 예약어 - #con 페이지 전체 영역 - #hd 머리글 영역 - #page1 본문 영역 - #content 주요 콘텐츠 영역 - #ft 바닥글 영역

3. 팝업 레이아웃 예약어 - 레이아웃 예약어 앞에 'pop_'를 조합하여 사용한다.

3. 이미지 네이밍 규칙

1. 같은 분류의 이미지가 두 개 이상이면 파일 이름 마지막에 숫자를 추가하여 구분한다.

2. HTML 코드 작성 규칙

1. 기본 규칙

1. W3C Validation을 통과해야 한다.

2. DTD를 제외한 모든 엘리먼트와 애트리뷰트는 소문자로 작성한다.

3. 애트리뷰트값은 큰따옴표(" ")로 묶는다.

2. 들여쓰기 규칙

1. 마크업의 중첩이 깊어질 때마다 자식 엘리먼트는 1탭 들여 쓴다.

2. 반드시 탭을 이용하여 들여쓰기를 한다.

3. 탭을 대신하여 공백으로 띄어 쓰지 않는다.

3. DTD 및 인코딩

1. DTD 선언 : HTML5

2. 인코딩 선언 : <meta charset="utf-8">

4. 주석

1. <!-- 주석 내용 -->

3. HTML 엘리먼트 작성 규칙

1. 전역 구조화 엘리먼트

1. <html> : <html lang="ko">

2. <head> : meta, title, link, script, style 순서로 엘리먼트를 선언한다.

3. <meta> : 문서의 기본 인코딩, 뷰포트, 스크립트 형식, 스타일 형식 순서로 선언한다.

뷰포트는 모바일 브라우저에 대응하는 HTML의 경우에만 선언한다.

4. <link> : rel, type, href 순서로 애트리뷰트를 선언한다.

5. <script> : type, src 순서로 애트리뷰트를 선언한다.

6. <style> : type 애트리뷰트를 선언한다.

2. 표 엘리먼트

1. <table> : 표의 요약 내용을 표기해야 할 때 summary 애트리뷰트를 할 수 있다.

2. <caption> : 표의 제목을 표현하기 위해 사용한다.

3. <col> : 표 각 열의 너비 지정을 위해 선언한다.

4. <thead> : 표 머리글을 그루핑할 때 선언한다.

5. <tfoot> : 표 바닥글을 그루핑할 때 선언한다.

tfoot 엘리먼트는 thead와 tbody 엘리먼트 사이에 위치해야 한다.

6. <tbody> : 표 본문을 그루핑하기 위해 선언한다.

3. 기타 엘리먼트

1. <a> : 새 창으로 페이지를 표시해야 할 때 target 애트리뷰트를 사용한다.

2. <img> : 이미지 내용과 동일한 값을 alt 애트리뷰트에 표기하여 이미지를 볼 수 없는 환경에서도 내용을 확인할 수 있게 한다.

3. title 애트리뷰트는 alt 애트리뷰트값을 축약하거나 브라우저에 독립적으로 툴팁을 표현하기 위해 사용한다.

4. CSS 코드 작성 규칙

1. 기본 규칙

1. 영문 소문자 사용

2. 따옴표 사용 범위

- 공백이 포함된 폰트명, 한글 폰트명, 문자열 데이터 타입은 작은따옴표(' ')로 감싸며, @charset 선언 시에는 속성 값을 큰따옴표(" ")로 감싼다. - url 데이터 타입에는 작은 따옴표를 사용하지 않는다.

2. 들여쓰기

1. CSS 코드를 작성할 때는 들여쓰기를 하지 않는다.

3. 빈 줄

1. 의미 있는 객체를 구분하기 위하여 코드 그룹 각 1줄씩 빈 줄을 넣을 수 있다.

2. 단, 빈 줄의 간격은 1줄을 초과하지 않게 한다.

4. 줄 바꿈

1. 선택자, 속성, 속성 값 사이 줄 바꿈은 허용하지 않는다.

5. 인코딩

1. HTML과 동일한 인코딩을 문서 첫 줄에 공백 없이 선언한다.

6. 속성

1. 약식 속성 사용 범위

- border : 속성 값은 border-width, border-style, border-color 순서로 선언한다. - font : 폰트 스타일은 약식 속성으로 선언하지 않는다. - 한글 폰트 선언 : 한글 폰트 선언 시 한글, 영문 폰트를 모두 선언한다.

7. CSS 선언 타입

1. External type : link 엘리먼트를 통해 HTML과 CSS 파일을 연결한다.

2. Internal(embedded) type : 페이지의 CSS 분량이 작을 경우 사용한다.

3. Inline type : 속성 값이 동적으로 변경되어야 하는 경우 사용한다.

8. 주석

1. /* 주석 내용 */

5. 웹 접근성 보장 방법

1. 의미에 맞는 마크업

1. 적절한 헤딩 엘리먼트 사용

: h1~h6의 헤딩 엘리먼트를 순차적으로, 적절한 위치에 사용한다.

2. 논리적인 순서 보장

1. 문서의 흐름과 동일하게 마크업 순서를 결정한다.

3. 대체 텍스트 제공

1. 이미지 대체 텍스트 제공

- 이미지 내용과 동일한 값을 alt 애트리뷰트에 표기하여, 이미지를 볼 수 없는 환경에서도 내용을 확인할 수 있게 한다.

2. 원문 그대로의 대체 텍스트 제공

- 이미지에 포함된 내용 그대로 대체 텍스트를 제공한다.

3. 텍스트가 없는 정보성 이미지의 대체 텍스트 제공

- 이미지가 나타내는 정보를 alt 애트리뷰트값에 표기한다. - 부연설명을 제공하는 컨텐츠가 있는 경우 대체 텍스트를 제공하지 않는다.

4. 표의 구성

1. <caption>

- 표의 제목을 나타내기위해 사용하며, <table> 태그 바로 아래 선언한다.

2. <thead>, <tfoot>, <tbody> 엘리먼트 사용

- <thead>,<tfoot>, <tbody> 엘리먼트를 사용하여 머리글과 바닥글을 분리한다.

5. 온라인 서식 구성

1. <fieldset> 을 이용한 그루핑

: 제목(캡션) 역할을 하는 값을 <legend> 안에 명시하여 접근성을 높이도록 한다.

6. 키보드 접근성 보장

1. 장치 독립적인 이벤트 사용

- onClick, onFocus, onMouseOut, onMouseUp, onMouseDown, onMouseOver

7. 배경 색상 제공(모바일)

1. 이미지를 불러오지 못하거나 느린 네트웍 속도로 인해 이미지를 빠르게 가져오지 못하는 경우에도 콘텐츠를 이용할 수 있도록 배경 색상을 제공한다.
